The core of this evolution relies on customized **Nanobot**, a highly capable personal AI agent running locally via Ollama, and orchestrated via **Telegram**.

This post explores the full architecture of how OpenMemo functions as an automated cognitive system, allowing me to build, deploy, and interact with my knowledge base effortlessly.


### 1. Telegram as the Unified Interface

I chose Telegram as the sole interaction channel for Nanobot. Since Telegram is mobile-first and deeply integrated into my daily routine, I can issue commands to Nanobot whether I'm at my desk or on the go. As seen in the flowchart above, the output stream is completely seamless.

### 2. Local Nanobot Execution

Nanobot represents the underlying backend intelligence safely running on local resources. I have tried openclawd but it is not as secure as nanobot. By harnessing local LLMs through Ollama, I maintain absolute command over the operational privacy and computing costs. Nanobot listens constantly, interpreting my Telegram tasks into a localized step-by-step logic loop. It can search the web, discover new ideas, and even create new blog posts and deploy them to the cloud.

### 3. Orchestrating the OpenMemo Webapp via Custom Skills

The true power of this system is how the local loop interacts with the globally accessible `open_memo` web app repository. I built **customized DevOps skills** into Nanobot that allow it to read, modify, and deploy code completely autonomously.

When I ask Nanobot to create a new blog or update my mind maps, the system:
1. Gathers context using local search and file reading skills.
2. Writes the `.mdx` content or modifies `React` components.
3. Automatically commits the changes and pushes directly to the remote GitHub repository.
4. **GCP Cloud Run** triggers a redeploy on the new push, instantly updating the live webapp at `https://kentchiu-memo-662008816033.asia-southeast1.run.app/`.

This means new knowledge goes from an idea in a Telegram chat to a live, interactive web page on GCP with zero manual intervention or API configuration on my part.
