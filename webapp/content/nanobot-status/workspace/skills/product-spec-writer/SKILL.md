# Product Spec Writer Skill

## Purpose
Create comprehensive product specifications for web, mobile, and desktop applications with user requirements, functional specs, technical architecture, UI/UX guidance, and success metrics.

## Overview
This skill helps you create detailed product specifications following industry best practices. It supports multiple output formats and includes templates for different app types.

## How to Use

1. **Tell me what product you want to spec**:
   - Product type (web app, mobile app, SaaS platform, etc.)
   - Key features and requirements
   - Target audience
   - Any specific constraints or preferences

2. **I'll automatically**:
   - Select the appropriate template
   - Fill in requirements, user stories, and technical specs
   - Validate for completeness and consistency
   - Output a comprehensive product specification

## Available Templates

| Template | Use Case |
|----------|----------|
| `templates/web-app.md` | Single-page or multi-page web applications |
| `templates/mobile-app.md` | iOS, Android, or cross-platform mobile apps |
| `templates/saas-platform.md` | SaaS applications with multi-tenancy |

## Output Formats

- **Markdown** - Human-readable documentation
- **JSON** - Machine-readable for CI/CD integration
- **Confluence-compatible** - For enterprise documentation

## Example Usage

```
Create a product spec for a project management SaaS platform with team collaboration features, timeline view, and integrations with Slack and GitHub.
```

## Best Practices

1. Always start with user needs and pain points
2. Include acceptance criteria for each requirement
3. Define success metrics and KPIs
4. Consider security and compliance requirements
5. Document technical constraints and dependencies

## Validation Checklist

- [ ] User stories cover all key features
- [ ] Acceptance criteria are clear and testable
- [ ] Technical architecture addresses scalability
- [ ] Security requirements are documented
- [ ] Success metrics are defined and measurable
